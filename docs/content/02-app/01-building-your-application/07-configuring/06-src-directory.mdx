除了在项目根目录中使用特殊的 Next.js`app` 或 `pages`目录外，Next.js 还支持将应用代码放置在 src 目录下的常见模式。

这种方式将应用代码与项目的配置文件分开，后者通常位于项目的根目录，这样的结构受到某些开发者和团队的青睐。

要使用`src`目录，只需将 app 路由文件夹或 pages 路由文件夹分别移动到 src/app 或 src/pages。

> **您需要知道**
>
> - `/public`目录应保留在项目的根目录。
> - 像`package.json`、`next.config.js`和`tsconfig.json`等配置文件应保留在项目根目录。
> - `.env.*`文件应保留在项目根目录。
> - 如果根目录下存在`app` 或 `pages`，则 `src/app` 或`src/pages`将被忽略。
> - 使用`src`目录时，通常也会将其他应用文件夹如 `/components` 或 `/lib` 移动到 `src` 下。
> - 如果使用中间件（Middleware），确保它位于`src`目录中。
> - 如果使用 Tailwind CSS，则需要在[content](https://tailwindcss.com/docs/content-configuration)配置部分的`tailwind.config.js`文件中添加`/src`前缀。
> - 如果使用 TypeScript 路径别名（如 @/\*），需要更新 `tsconfig.json` 中 `paths` 对象的配置以包含 `src/` 路径。
