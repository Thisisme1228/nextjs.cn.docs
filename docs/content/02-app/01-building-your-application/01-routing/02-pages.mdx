页面是路由**唯一的**UI。您可以通过在一个`page.js`文件中定义一个默认输出的组件来生成页面。

例如, 在`app`目录中添加一个`page.js`文件来创建`index`页面 ：

<img
  alt="page.js special file"
  src="/page-special-file.avif"
  width="100%"
  height="auto"
/>

```tsx filename="app/page.tsx" switcher
// `app/page.tsx` is the UI for the `/` URL
export default function Page() {
  return <h1>Hello, Home page!</h1>;
}
```

然后, 通过创建一个新文件夹并添加`page.js`文件在其中来创建更多的页面。例如：你能够创建一个新的`dashboard`文件夹，并且创建一个`page.js`文件在其文件夹中，来创建一个`/dashboard`路由。

```tsx filename="app/dashboard/page.tsx" switcher
// `app/dashboard/page.tsx` is the UI for the `/dashboard` URL
export default function Page() {
  return <h1>Hello, Dashboard Page!</h1>;
}
```

> **您需要知道**:
>
> - 页面能够使用 `.js`, `.jsx`, or `.tsx`文件扩展名。
> - 页面总是[路由子树](/docs/content/app/building-your-application/routing#terminology)的[叶子节点](/docs/content/app/building-your-application/routing#terminology)。
> - 需要一个`page.js`文件使路由段可公开访问。
> - 默认情况下，页面是[服务器组件](/docs/content/app/building-your-application/rendering/server-components), 但可以设置为[客户端组件](/docs/content/app/building-your-application/rendering/client-components).
> - 页面能够获取数据。查看[数据获取](/docs/content/app/building-your-application/data-fetching) 部分来了解更多。
